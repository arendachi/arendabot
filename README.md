# Telegram-бот: каталог объявлений недвижимости

Бот: `https://t.me/arendachiii`

Бот для поиска объявлений с фильтрацией по 7 категориям:

1. **Регион** (12 районов Ташкента)
2. **Количество комнат**
3. **Этаж**
4. **Этажность дома**
5. **Площадь**
6. **Цена** (в долларах, $300–$3000)
7. **Тип недвижимости** — Жилой / Нежилой

На шагах 2–6 есть кнопка **«✏️ Своё число»** — можно ввести точное значение вместо выбора диапазона (например, точно 7 комнат или точно 54 м²).

Также бот умеет:
- **Автоматически добавлять объявления**: админ присылает текст в формате JoyEstate (просто сообщением) — бот распознаёт поля, сохраняет в базу и публикует пост в канал.
- **Поиск объявления по номеру (ID)** — доступен всем: просто отправьте боту число.
- **Приватные данные владельца** — видны только админу. При поиске по ID админ видит кнопку «✏️ Редактировать владельца», куда можно вписать имя и телефон владельца квартиры. Эта информация никогда не публикуется в канале и не видна обычным пользователям.

## Структура проекта

```
realty_bot/
├── bot.py            # основная логика бота, диалог, клавиатуры, автодобавление, поиск по ID
├── database.py       # работа с SQLite (хранение и поиск объявлений)
├── config.py         # токен, админы, канал, справочники категорий
├── parser.py         # парсер текста объявлений формата JoyEstate
└── requirements.txt
```

```bash

```

Узнать свой Telegram ID можно у бота [@userinfobot](https://t.me/userinfobot).

**Важно:** бот должен быть добавлен в канал `CHANNEL_USERNAME` как администратор с правом публикации сообщений.

## Запуск

```bash
python bot.py
```

При первом запуске создаётся `realty.db` (SQLite) с тестовыми объявлениями. Если уже есть база со старой схемой — при запуске бот автоматически добавит новые колонки (`floors_total`, `area`, `source_listing_id`, `owner_name`, `owner_phone`), **не теряя существующие данные**.

Чтобы убрать тестовые данные — удалите вызов `db.seed_demo_data()` в `bot.py`.

## Как работает поиск по фильтрам

`/start` запускает диалог из 7 шагов с inline-кнопками. На шагах «Комнаты», «Этаж», «Этажность», «Площадь», «Цена» доступна кнопка **«✏️ Своё число»** — бот попросит написать точное число сообщением и продолжит диалог дальше. Можно вернуться назад («⬅ Назад») или выбрать «любое». После последнего шага — список подходящих объявлений с пагинацией.

## Как работает поиск по ID

Любой пользователь может отправить боту число (например `19`) — бот найдёт объявление с этим номером и покажет карточку в формате:

```
⚜️ Район: Мирзо Улугбек
🏬 Адрес: ЖК "Perfect house"
📍 Ориентир: Экобазар

🔻 Комнат: 2
🔻 Этаж: 2
🏢 Этажность: 5
📐 Площадь: 40 м²
🏷 Тип: 🏠 Жилой

💰 Цена: $600

🆔 #19

📞 Администратор: +998200100052
👤 Telegram: @ENTRNCEE
```

Под карточкой — кнопка **«Связаться с администратором и запросить фото»** (видна всем). При нажатии бот пересылает карточку объявления вместе с username и Telegram ID клиента всем админам из `ADMIN_IDS`, чтобы админ знал, кому отправить фотографии дома.

Если пользователь — админ, под карточкой дополнительно появляется кнопка **«✏️ Редактировать владельца»**.

Контакты администратора (`ADMIN_PHONE`, `ADMIN_USERNAME` в `config.py`) показываются в каждой карточке — это видно всем пользователям.

## Как работает добавление фотографий (только админ)

При поиске по ID админ видит кнопку **«📷 Добавить фото»** рядом с «Редактировать владельца».

1. Админ нажимает «📷 Добавить фото».
2. Бот просит прислать до 10 фотографий.
3. Админ присылает фото одно за другим (как обычные сообщения с фото).
4. Приём завершается:
   - **автоматически** после 10-й фотографии, или
   - **по команде `/done`**, если фотографий меньше 10.
5. Все новые фотографии **полностью заменяют** старые (если они были загружены ранее) — старые удаляются из базы.

Фотографии хранятся в базе как `file_id` от Telegram (не сами файлы) — это стандартный и лёгкий способ, не требующий отдельного хранилища файлов.

**Важно:** фотографии никогда не отправляются автоматически пользователям бота — ни при поиске, ни при нажатии «Связаться с администратором и запросить фото». Только админ может вручную переслать фото клиенту после того, как получит уведомление о запросе.

## Как посмотреть загруженные фотографии и управлять их видимостью

При поиске по ID, если у объявления есть фото, появляется кнопка **«🖼 Посмотреть фото»**:
- **Админу** она видна всегда (пока есть хотя бы одно фото).
- **Обычным пользователям** она видна только если админ сделал фото публичными.

По умолчанию все фото **приватны** — видны только админу.

Рядом с кнопками для админа есть переключатель:
- **«👁 Показать фото пользователям»** — делает фото видимыми всем.
- **«🙈 Скрыть фото от пользователей»** — возвращает фото в приватный режим.

Админ может переключать видимость в любой момент, сколько угодно раз. После переключения карточка обновляется на месте с новым набором кнопок.

Нажатие «🖼 Посмотреть фото» отправляет все фотографии в чат: одно фото — как обычное изображение, несколько — единым альбомом (media group).

## Как работает редактирование владельца (только админ)

1. Админ находит объявление по ID, нажимает «✏️ Редактировать владельца».
2. Бот спрашивает имя владельца — админ отправляет сообщением.
3. Бот спрашивает номер телефона — админ отправляет сообщением.
4. Данные сохраняются в базе в полях `owner_name` / `owner_phone`.

Эта информация:
- видна **только** при поиске по ID, и только админу (обычные пользователи никогда её не увидят, даже в результатах обычного поиска по фильтрам);
- **не влияет** на автопубликацию из JoyEstate и не попадает в посты канала — это полностью отдельные приватные поля.

## Как работает автодобавление объявлений из JoyEstate

1. Админ пересылает или вставляет в чат с ботом текст вида:

```
JoyEstate
Краткое описание:

⚜️  Pайон: chilanzar
🏬 Адрес: Чиланзар 3 квартал
📍 Ориентир: Чиланзар 3 квартал

Информация о квартиры:
🔻 Комнат: 2
🔻 Этаж: 2
🔻 Этажность: 4
🔻 Площадь: 41 m²

💰 Цена: $600.00

🆔: 19487
```

2. Бот узнаёт формат по ключевым подписям и парсит его регулярками (без платных AI API).
3. Распознанное объявление сохраняется в базу.
4. Готовый пост автоматически публикуется в `CHANNEL_USERNAME` — без подтверждения.
5. Если объявление с таким же `🆔` уже было добавлено — бот не публикует дубликат.
6. Если обязательные поля не распознаны — бот сообщает, что именно не нашёл, и не публикует ничего.
7. Обычные пользователи (не из `ADMIN_IDS`) не могут использовать эту функцию.

### Если формат источника поменяется

Парсер (`parser.py`) основан на регулярных выражениях. Он устойчив к небольшим вариациям, но если источник изменит сам набор полей — нужно обновить регулярки в `parser.py`. Альтернатива при частых изменениях формата — заменить парсер на запрос к Claude API (платный за каждый запрос, но надёжнее к изменениям).

## Настройка категорий

Все справочники редактируются в `config.py`:

- `REGIONS` — 12 районов Ташкента
- `DISTRICT_ALIASES` — сопоставление названий районов из источника JoyEstate (например, `"chilanzar"`) с читаемыми названиями (`"Чиланзар"`)
- `ROOM_OPTIONS` — варианты количества комнат (плюс кнопка «своё число» в боте)
- `FLOOR_RANGES`, `FLOORS_TOTAL_RANGES`, `AREA_RANGES` — диапазоны для соответствующих фильтров
- `PRICE_RANGES` — диапазоны цен в долларах ($300–$3000 и шире)
- `PROPERTY_TYPES` — типы недвижимости
- `ADMIN_IDS` — кто может присылать тексты для автодобавления и редактировать владельцев
- `CHANNEL_USERNAME` — куда публиковать объявления

```python
import database as db
db.init_db()
db.add_listing(
    region="Чиланзар",
    rooms=2,
    floor=5,
    price=450,
    property_type="residential",
    floors_total=9,
    area=54.0,
    title="2-комнатная квартира с видом на парк",
    description="Свежий ремонт, мебель",
)
# Добавить владельца (видно только админу):
db.update_owner_info(listing_id=1, owner_name="Алишер", owner_phone="+998901234567")
```
