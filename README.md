# Telegram-бот: каталог объявлений недвижимости

Бот: `https://t.me/arendachiii`

Бот для поиска объявлений с фильтрацией по 7 категориям:

1. **Регион**
2. **Количество комнат**
3. **Этаж**
4. **Этажность дома**
5. **Площадь**
6. **Цена**
7. **Тип недвижимости** — Жилой / Нежилой

А также умеет **автоматически добавлять объявления**: админ присылает боту текст в формате JoyEstate (просто как сообщение, без команды) — бот распознаёт поля, сохраняет в базу и публикует готовый пост в Telegram-канал.

## Структура проекта

```
realty_bot/
├── bot.py            # основная логика бота, диалог, клавиатуры, автодобавление
├── database.py       # работа с SQLite (хранение и поиск объявлений)
├── config.py         # токен, админы, канал, справочники категорий
├── parser.py         # парсер текста объявлений формата JoyEstate
└── requirements.txt
```

## Установка

```bash
pip install -r requirements.txt
```

## Настройка

Все параметры задаются через переменные окружения (или прямо в `config.py`):

```bash
export BOT_TOKEN="123456:ABC-ваш-токен"        # токен от @BotFather
export ADMIN_IDS="123456789,987654321"          # ID админов, разделённые запятой
export CHANNEL_USERNAME="@your_channel"         # канал для автопубликации объявлений
```

Узнать свой Telegram ID можно у бота [@userinfobot](https://t.me/userinfobot).

**Важно:** бот должен быть добавлен в канал `CHANNEL_USERNAME` как администратор с правом публикации сообщений — иначе автопубликация не будет работать (придёт предупреждение об ошибке прямо в чат с ботом).

## Запуск

```bash
python bot.py
```

При первом запуске автоматически создаётся `realty.db` (SQLite) и заполняется 10 тестовыми объявлениями. Если уже есть старая база без новых колонок (`floors_total`, `area`) — при запуске бот сам выполнит миграцию и добавит их, **не потеряв существующие данные**.

Чтобы убрать тестовые данные, удалите вызов `db.seed_demo_data()` в `bot.py`.

## Как работает поиск

`/start` запускает пошаговый диалог из 7 шагов с inline-кнопками: регион → комнаты → этаж → этажность → площадь → цена → тип. На любом шаге можно выбрать «любое» или вернуться назад («⬅ Назад»). После последнего шага — список подходящих объявлений с пагинацией.

## Как работает автодобавление объявлений

1. Админ (ID из `ADMIN_IDS`) просто пересылает или вставляет в чат с ботом текст в формате:

```
JoyEstate
Краткое описание:

⚜️  Pайон: chilanzar
🏬 Адрес: Чиланзар 3 квартал
📍 Ориентир: Чиланзар 3 квартал

Информация о квартиры:
🔻 Комнат: 2
🔻 Этаж: 2
🔻 Этажность: 4
🔻 Площадь: 41 m²

💰 Цена: $600.00

🆔: 19487
```

2. Бот узнаёт формат по ключевым подписям (район, комнаты, этаж, цена) и парсит его регулярками — без обращения к каким-либо платным AI API.
3. Распознанное объявление сразу сохраняется в базу.
4. Готовый пост автоматически публикуется в `CHANNEL_USERNAME` — без дополнительного подтверждения.
5. Если объявление с таким же `🆔` уже было добавлено ранее — бот не публикует дубликат и сообщает об этом админу.
6. Если обязательные поля (район, комнаты, этаж, цена) не распознаны — бот пишет, что именно не нашёл, и не публикует ничего.
7. Обычные пользователи (не из `ADMIN_IDS`) не могут использовать эту функцию — их текстовые сообщения боту полностью игнорируются этим механизмом.

### Если формат источника поменяется

Парсер (`parser.py`) основан на регулярных выражениях, которые ищут подписи вида `Район:`, `Комнат:`, `Этаж:` и т.д. независимо от эмодзи перед ними. Он устойчив к небольшим вариациям (лишние пробелы, "Pайон" с латинской P, "кв.м" вместо "m²"), но если источник изменит сам набор полей или формулировки подписей — потребуется обновить регулярки в `parser.py`. Если такое происходит часто, можно заменить парсер на запрос к Claude API (через `anthropic` SDK) — он будет надёжнее к изменениям формата, но платный за каждый запрос.

## Настройка категорий

Все справочники редактируются в `config.py`:

- `REGIONS` — список регионов/городов для ручного поиска
- `DISTRICT_ALIASES` — сопоставление названий районов из источника (например, `"chilanzar"`) с читаемыми названиями (`"Чиланзар"`)
- `ROOM_OPTIONS` — варианты количества комнат
- `FLOOR_RANGES` — диапазоны этажей
- `FLOORS_TOTAL_RANGES` — диапазоны этажности дома
- `AREA_RANGES` — диапазоны площади (м²)
- `PRICE_RANGES` — диапазоны цен (`None` означает «без ограничения»)
- `PROPERTY_TYPES` — типы недвижимости
- `ADMIN_IDS` — кто может присылать тексты для автодобавления
- `CHANNEL_USERNAME` — куда публиковать объявления

```python

db.init_db()
db.add_listing(
    region="Ташкент",
    rooms=2,
    floor=5,
    price=55000,
    property_type="residential",
    floors_total=9,
    area=54.0,
    title="2-комнатная квартира с видом на парк",
    description="Свежий ремонт, мебель",
    contact="@username или +998..."
)
```
